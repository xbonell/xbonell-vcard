# Product Context

This document captures non-operational context that helps agents and contributors
understand why this site exists and what it should optimize for.

## Product Vision

`xbonell-vcard` is a personal vCard site for `xbonell.com`.
It should feel professional, fast, and intentional while staying lightweight and easy to maintain.

## Audience

- Primary: potential clients, hiring managers, and professional contacts.
- Secondary: peers in the web development community.
- Scope: multilingual audience (English, Spanish, Catalan).

## Core Value Proposition

- Present professional identity clearly and credibly.
- Make contact paths obvious and low friction.
- Showcase current frontend specialization (React, Vue.js, WordPress).
- Deliver excellent performance and accessibility.

## Product Principles

- Keep content concise and useful.
- Prioritize readability over decoration.
- Preserve multilingual parity for key sections.
- Prefer stable, boring infrastructure over unnecessary complexity.

## Key UX Outcomes

- Fast first load and smooth interactions.
- Strong legibility across desktop and mobile breakpoints.
- Accessible markup and behavior (semantic HTML, proper labels, ARIA where needed).
- Consistent visual identity across languages.

## Feature Set (Current)

- Multilingual content (`en`, `es`, `ca`).
- Theme system with `light` / `dark` / `system` modes.
- Professional profile, services, toolkit, and social links.
- Interactive hole/logo viewer effect in the page background.
- Error page with dynamic status code rendering.
- Static output suitable for simple hosting.

## Content Priorities

- Accurate role and specialization messaging.
- Up-to-date toolkit/services entries.
- Contact info and social links kept current.
- SEO metadata maintained for all language variants.

## Non-goals

- Turning this into a complex web app.
- Introducing backend runtime dependencies.
- Adding heavy frameworks or tooling without clear value.

## Success Signals

- Site builds cleanly and deploys consistently.
- No regressions in multilingual navigation/content parity.
- No regressions in mobile layout or accessibility basics.
- Changes remain aligned with a fast, minimalist professional profile.
